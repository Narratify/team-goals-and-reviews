# 評価面談 実践スクリプト&テンプレート集

## 評価面談シナリオ別スクリプト

### シナリオ1: 優秀な成果を伝える場合

```markdown
【オープニング】
「○○さん、今期もお疲れ様でした。今日は8期下期の評価についてお話ししたいと思います。
まず最初に言いたいのは、あなたの貢献に本当に感謝しているということです。」

【自己評価の確認】
「まず、○○さん自身はこの半期をどう振り返っていますか？
特に印象に残っている成果や挑戦はありましたか？」

【評価の伝達】
「総合評価はAで、期待を大きく上回る成果でした。特に3つの点で卓越していました：

1. ITOKIプロジェクトでの技術的ブレークスルー
   - visionOS問題を24時間以内に解決
   - これによりプロジェクトの遅延を防ぎ、顧客信頼を維持

2. チームへの貢献
   - 週4-6件のペースで高品質アセットを制作
   - 他メンバーの技術的な質問にも積極的にサポート

3. 顧客満足度
   - NHK技研から『5段階評価で6をつけたい』という異例の高評価
   - これは私たちのチーム全体の評判を高めました」

【今後への期待】
「次期は、この素晴らしい技術力を活かして、より戦略的な役割も担っていただきたいです。
例えば、技術選定や若手メンバーのメンタリングなどはいかがでしょうか？」

【クロージング】
「何か質問や、私からのサポートが必要なことはありますか？
改めて、素晴らしい成果をありがとうございました。」
```

### シナリオ2: 改善が必要な点を伝える場合

```markdown
【オープニング - 心理的安全性の確保】
「○○さん、まず最初に伝えたいのは、この評価面談の目的は
あなたの成長をサポートすることだということです。
一緒に改善点を見つけて、次期はより良い成果を出せるようにしましょう。」

【自己認識の確認】
「この半期を振り返って、ご自身で『もっとこうすればよかった』
と思う点はありますか？」

【改善点の伝達 - SBIモデル使用】
「私からのフィードバックをお伝えします：

Situation: 7月と8月のプロジェクトで
Behavior: 納期を計3回延期することになりました
Impact: これによりチーム全体のスケジュールに影響が出て、
        他のメンバーの負担が増加しました

ただ、これは能力の問題ではなく、タスク管理の方法を
改善することで解決できると考えています。」

【具体的な改善案の提示】
「改善案として3つの選択肢を考えました：
1. 週次での進捗確認ミーティング
2. タスク管理ツールの活用トレーニング
3. ペアプログラミングでの実践的学習

どれが最も効果的だと思いますか？」

【サポートの約束】
「私も全力でサポートします。週1回の1on1を設定して、
進捗確認と必要な支援を提供します。
3ヶ月後には必ず改善が見られると信じています。」

【前向きなクロージング】
「あなたには確実に成長のポテンシャルがあります。
一緒に頑張りましょう。」
```

### シナリオ3: 昇格・昇給を伝える場合

```markdown
【オープニング - 良いニュースの予告】
「○○さん、今日は嬉しいお知らせがあります。
あなたの継続的な成長と貢献が認められました。」

【具体的な評価内容】
「8期下期の評価結果として：
- Grade 4から5への昇格が決定しました
- 月給は20,000円増額の460,000円となります

この決定の背景には：
1. visionOS等の新技術への迅速な対応力
2. 複数プロジェクトでの安定した高品質アウトプット
3. Tokyo Nodeハッカソンでの選出（200組中15組）
これらの実績が高く評価されました。」

【新しい責任と期待】
「Grade 5として、次期は以下を期待しています：
- より複雑なプロジェクトのリード
- 技術的な意思決定への参画
- ジュニアメンバーへのメンタリング

準備はできていますか？」

【条件とサポート】
「ただし、以下の2点は改善をお願いします：
1. 月間労働時間を160時間以内に管理
2. 勤怠打刻の100%実施

これらを支援するため、タスク配分の見直しも行います。」

【祝福と激励】
「本当におめでとうございます！
新しい役割でのさらなる活躍を期待しています。」
```

### シナリオ4: 厳しい評価（PIP）を伝える場合

```markdown
【事前準備の確認】
※必ず HR部門と相談し、法的要件を確認する
※記録を残すため、第三者の同席を検討

【オープニング - 真剣だが支援的なトーン】
「○○さん、今日は重要な話があります。
現在のパフォーマンスについて、率直にお話しする必要があります。
ただ、私たちの目的は、あなたが成功できるよう支援することです。」

【事実ベースの説明】
「具体的なデータをお示しします：
- 目標達成率: 45%（目標80%に対して）
- 品質指標: 要改善レベルが3ヶ月連続
- 顧客満足度: チーム平均を20ポイント下回る

これらは改善が必要な水準です。」

【Performance Improvement Plan の提示】
「そこで、今後3ヶ月間のPerformance Improvement Planを
一緒に作成したいと思います：

月次目標：
- 1ヶ月目: 基礎スキルの再確認と強化
- 2ヶ月目: 実践での応用と週次フィードバック
- 3ヶ月目: 自立的な業務遂行

サポート体制：
- 週2回のメンタリング
- 必要な研修への参加
- 明確な成功基準の設定」

【選択肢の提示】
「このプランにコミットしていただけますか？
もし他の役割の方が適していると感じるなら、
その可能性も一緒に探ることができます。」

【明確な結果の説明】
「3ヶ月後に明確な改善が見られない場合、
役割の変更や、場合によっては雇用の見直しも
検討する必要があることをご理解ください。
ただし、私は改善できると信じています。」
```

```markdown
✓「顧客から『○○さんのおかげで成功した』という言葉をいただきました」

【成長を認める】
✓「半年前と比べて、○○のスキルが格段に向上しています」
✓「困難な状況での対応力が確実に身についてきています」
✓「リーダーシップの芽が見えてきました」

【努力を認める】
✓「リソース制約の中でのあなたの努力を高く評価しています」

```

### 建設的フィードバック

```markdown
【改善を促す】
✓「○○の点で、さらなる成長の余地があります」
✓「次のレベルに到達するために、○○を強化しましょう」
✓「○○を改善することで、あなたの強みがより活きるはずです」

【具体的な指摘】
✓「たとえば、先週の○○の場面では、△△という対応がより効果的だったかもしれません」
✓「データを見ると、○○の部分で改善の機会があることがわかります」
✓「顧客からのフィードバックで、○○についての要望がありました」

【サポートを申し出る】
✓「○○を改善するために、どんなサポートが必要ですか？」
✓「一緒に改善計画を作成しましょう」
✓「○○のスキルアップのための研修を用意できます」
```

### 困難な状況での対話

```markdown
【感情的な反応への対処】
✓「お気持ちは理解できます。少し時間を取りましょうか？」
✓「あなたの frustration は当然だと思います」
✓「この評価が unexpected だったことは理解しています」

【抵抗への対応】
✓「あなたの視点を聞かせてください」
✓「私が見落としている点があるかもしれません」
✓「具体的にどの部分に同意できないか教えてください」

【建設的な方向への転換】
✓「では、どうすれば状況を改善できるでしょうか？」
✓「次期により良い評価を得るために何が必要でしょうか？」
✓「一緒に解決策を見つけましょう」
```

### 改善計画書（PIP）

```markdown
# Performance Improvement Plan (業績改善計画書)

対象者: ○○ ○○
期間: 2025年10月1日 - 2025年12月31日
作成日: 2025年9月25日

## 1. 現状の課題
- 納期遵守率: 60%（目標: 90%）
- 品質基準達成率: 70%（目標: 95%）
- コミュニケーション: 改善が必要

## 2. 改善目標
### 30日目標
- 納期遵守率 75%達成
- 週次進捗報告の100%実施
- 品質チェックリストの活用開始

### 60日目標
- 納期遵守率 85%達成
- 品質基準達成率 85%
- 能動的なコミュニケーション実践

### 90日目標
- 全指標で目標値達成
- 自律的な業務遂行
- チームへの貢献

## 3. サポート体制
- 週次1on1ミーティング（毎週月曜 10:00）
- メンター制度の活用
- 必要な研修への参加承認

## 4. 評価基準
- 定量指標の達成度
- 定性的な改善の観察
- 360度フィードバック

## 5. 結果と次のステップ
改善達成の場合: 通常評価へ復帰
改善不十分の場合: 役割変更の検討

署名:
対象者: _________________ 日付: _______
上司: ___________________ 日付: _______
HR: ____________________ 日付: _______
```

## オンライン評価面談の注意点

```markdown
【技術的準備】
□ 安定したインターネット接続
□ カメラ・マイクのテスト
□ 画面共有の準備
□ バックアップの通信手段

【環境設定】
□ 静かでプライベートな空間
□ 適切な照明
□ プロフェッショナルな背景
□ 通知をオフに

【コミュニケーションの工夫】
□ アイコンタクト（カメラを見る）
□ 明確な発話
□ 定期的な理解確認
□ 画面共有での視覚的説明

【特別な配慮】
□ 時差がある場合の配慮
□ 技術的トラブルへの対処法準備
□ 録画の可否確認
□ デジタル疲労への配慮
```
